# ネットワーク接続環境の設定ガイド

本システム（Citadel - 現場報告管理システム）の Scout（現場用 PWA）を、オフライン機能付きで快適に使うためには、**安全な接続（HTTPS）** でアクセスする必要があります。  
このドキュメントは、運用担当者の方が「なぜその設定が必要か」と「具体的に何をすればよいか」を理解できるようにまとめたものです。

---

## この設定が必要な理由

### ブラウザの「セキュアコンテキスト」とは

Chrome や Edge などのブラウザには、**セキュリティ保護機能** が組み込まれています。  
その一環として、次のような接続では「安全が確認できない」と判断し、一部の機能を**強制的に止める**仕様になっています。

- **http://** で始まるアドレス（暗号化されていない通信）
- **IP アドレス** を直接指定した接続（例：`http://100.64.0.1:3000`）
- Tailscale や社内 LAN などで **IP で開いている** 場合

このような接続を、ブラウザでは「**非セキュアなコンテキスト**」と呼び、次のような重要な機能が使えません。

| 機能             | 説明                                                 |
| ---------------- | ---------------------------------------------------- |
| オフライン起動   | ネットが切れてもアプリを開いたまま使う               |
| オフライン準備   | あらかじめページをキャッシュしてオフラインで表示する |
| 一部のデータ保存 | ブラウザ内での安全な保存（Service Worker 等）        |

そのため、

- 「オフラインで使いたい」
- 「現場でネットが不安定な環境でも使いたい」

といった運用をしたい場合は、**HTTPS で接続できる環境** を用意することが推奨されます。

---

## 推奨する解決策：HTTPS で接続する

### 何をすればよいか

**システムの管理者** に依頼し、接続先を **「https://」で始まる URL** に変更してもらってください。

- **変更前の例**：`http://100.64.0.1:3000` や `http://scout.local:3000`
- **変更後の例**：`https://scout.example.com` や `https://scout.自社ドメイン.co.jp`

こうすると、

1. 通信が暗号化され、ブラウザが「安全な接続」と判断する
2. オフライン機能（オフライン起動・オフライン準備）が利用可能になる
3. データのやり取りも保護される

というメリットがあります。

### 管理者向けの技術的なポイント

- Web サーバー（Nginx / Apache / Caddy など）で HTTPS を有効にする
- 証明書は、社内用であれば **Let's Encrypt** や **社内 CA** で発行した証明書を利用可能
- 本番起動手順は `docs/PRODUCTION.md` を参照

## 緊急時の回避策：ブラウザの「フラグ」で例外設定する

「すぐには HTTPS に切り替えられないが、今だけオフライン機能を使いたい」という場合に限り、Chrome または Edge の **フラグ（実験的設定）** で、**この接続先だけ** を「安全」と扱う設定ができます。

> **注意**  
> この方法は一時的な回避策です。セキュリティ上、可能な限り早めに **HTTPS 環境への移行** を検討してください。

### 対象ブラウザ

- **Google Chrome**
- **Microsoft Edge**

（いずれも最新版を推奨）

### 手順（Chrome の場合）

1. **フラグのページを開く**
   - アドレスバーに次の URL を**そのまま**入力して Enter を押す。
     ```
     chrome://flags/#unsafely-treat-insecure-origin-as-secure
     ```
   - Edge の場合は次のように入力する。
     ```
     edge://flags/#unsafely-treat-insecure-origin-as-secure
     ```

2. **該当する設定を探す**
   - ページ内で「**Insecure origins treated as secure**」という項目を探す。
   - 見つからない場合は、ページ内の検索ボックスに「insecure」と入力して絞り込む。

3. **接続先のアドレスを入力する**
   - その項目の右側にある **テキスト欄** に、**今使っている接続先のアドレス** を入力する。
   - 例：
     - `http://100.64.0.1:3000`
     - `http://scout.local:3000`
   - 複数ある場合はカンマで区切って並べてよい（例：`http://100.64.0.1:3000,http://scout.local:3000`）。
   - **https://** のアドレスは含めないこと。

4. **設定を有効にする**
   - 右端のドロップダウンで「**Enabled**」を選ぶ。

5. **ブラウザを再起動する**
   - 画面下部に「**Relaunch**」ボタンが表示されるので、クリックしてブラウザを再起動する。
   - 開いていたタブは再度開き直す必要があります。

6. **動作確認**
   - 再起動後、同じアドレス（例：`http://100.64.0.1:3000`）で Scout にアクセスし直す。
   - データ管理画面の「現在の接続環境ではオフライン機能が利用できません」という警告が消え、オフライン準備などが使えるようになっているか確認する。

### 手順（Edge の場合）

- 上記の「手順（Chrome の場合）」で、  
  `chrome://flags/...` の部分を **`edge://flags/...`** に読み替えて同じ操作を行う。
- 項目名「Insecure origins treated as secure」や、アドレス入力・Enabled 選択・Relaunch の流れは同じ。

### この設定をやめたい場合

1. 再度 `chrome://flags/#unsafely-treat-insecure-origin-as-secure`（または Edge の同等ページ）を開く。
2. 「Insecure origins treated as secure」を「**Disabled**」に戻す。
3. 「Relaunch」でブラウザを再起動する。

---

## まとめ

| 目的                                  | 推奨する対応                                                           |
| ------------------------------------- | ---------------------------------------------------------------------- |
| 通常運用・本番利用                    | **HTTPS** で接続できるようにする（管理者に依頼）                       |
| すぐに HTTPS にできない場合の一時対応 | 上記の **フラグ** で接続先を例外登録する（利用者・管理者の判断で実施） |

オフライン機能を安定して使うには、**HTTPS 環境の整備** が最も確実です。  
不明な点があれば、システム管理者または開発担当者に相談してください。
